# NNgram — игра Нонограммы (Picross)

Мобильная игра-головоломка **Нонограмм** (Picross) на Flutter.  
Игрок заполняет клетки сетки по числовым подсказкам, расположенным по краям поля, чтобы получить скрытую картинку.

---

## Обзор проекта

- **Что это**: офлайн-игра-головоломка Нонограммы для мобильных устройств.
- **Цель продукта**: коллекция уровней с разной сложностью, сохранение прогресса и простой, понятный интерфейс.
- **Целевая аудитория**: люди, любящие логические пазлы и неторопливые интеллектуальные игры.

---

## Игровая механика (кратко)

- Есть прямоугольная сетка клеток.
- Слева от строк и сверху от столбцов указаны **подсказки‑числа**.
- Каждое число — длина непрерывной группы закрашенных клеток.
  - Пример: `[3, 1]` — три закрашенные клетки подряд, хотя бы одна пустая, затем ещё одна закрашенная.
  - `[0]` — в строке/столбце нет ни одной закрашенной клетки.
- Победа: фактический рисунок игрока полностью соответствует подсказкам (все закрашенные клетки совпадают с решением).

Режимы взаимодействия:

- **Fill** — закрасить/отменить закрашивание клетки.
- **Cross** — поставить/убрать крестик для пометки «точно пусто».

## Технологический стек

- **Flutter 3.x**, **Dart >= 3.0**
- **Состояние**: `flutter_riverpod`
- **Навигация**: `go_router`
- **База данных**: `drift` + `drift_flutter` + `sqlite3_flutter_libs`
- **Модели и JSON**: `freezed`, `json_serializable`

Точные версии см. в `pubspec.yaml`.

## Как запустить проект (для разработчика)

Ниже — базовые шаги для разработки и запуска.

### 1. Требования

- Установленный **Flutter SDK** (3.x, Dart >= 3.0) и добавленный в `PATH`.
- Настроенное устройство для запуска:
  - либо **Android-эмулятор** (через Android Studio или командные инструменты),
  - либо **реальное Android-устройство** с включённым USB‑отладчиком.
- Любой редактор кода, например **VS Code**. Android Studio *не обязателен*, если у вас уже есть устройство для запуска и нужные Android SDK/драйверы.

Проверка окружения:

```bash
flutter doctor
```

Убедитесь, что нет критических ошибок по Flutter и Android toolchain.

### 2. Установка зависимостей

В корне проекта:

```bash
flutter pub get
```

### 3. Генерация кода

Кодогенерация для моделей (`freezed`, `json_serializable`) и базы данных (`drift`):

```bash
flutter pub run build_runner build --delete-conflicting-outputs
```

Эту команду нужно повторять после изменений в моделях/таблицах.

### 4. Запуск приложения

1. Подключите реальное Android‑устройство по USB **или** запустите эмулятор.
2. Проверьте наличие устройства:

```bash
flutter devices
```

3. Запустите приложение:

```bash
flutter run
```

Flutter соберёт и установит приложение на выбранное устройство.

Для релизного запуска (когда всё стабильно):

```bash
flutter run --release
```
